# Global Domain Dictionaries

> 이 폴더는 사용자(당신)의 **도메인 지식 자산**입니다.
> 여러 프로젝트에서 누적된 도메인 용어를 참고용 씨앗으로 보관합니다.

## 폴더 구조

```
~/.claude/memory/domain-dictionaries/
├── README.md          ← 이 파일
├── ecommerce.md       ← 이커머스
├── healthcare.md      ← 의료
├── finance.md         ← 금융
├── education.md       ← 교육
├── logistics.md       ← 물류/SCM
├── manufacturing.md   ← 제조
├── realestate.md      ← 부동산
└── general.md         ← 도메인 무관 (인증, 세션, 감사 등)
```

도메인 파일들은 빈 상태로 시작합니다. 첫 프로젝트 작업이 끝날 때 사용자가 `Step 11`에서 글로벌 반영을 선택하면 자동으로 채워집니다.

## 작동 방식

### 새 프로젝트 시작 (zephermine 또는 직접 호출)
1. 도메인 자동 추정 (인터뷰의 `[Industry]` 태그 또는 사용자 선택)
2. 해당 도메인 파일이 비어있지 않으면 → 후보 용어 multiSelect 제시
3. 사용자가 ✅한 용어만 프로젝트 사전 v1 시드로 사용

### 프로젝트 종료 (zephermine Step 11)
1. 이번 프로젝트에서 새로 정의/다듬어진 용어 추출
2. multiSelect로 글로벌 반영 선택지 제시
3. 사용자가 ✅한 항목만 글로벌 도메인 파일에 추가
4. 출처 메타데이터 함께 기록 (어느 프로젝트, 언제, 확신도)

## 용어 형식

```markdown
### Cart
**정의**: 결제 전 임시로 모은 상품 목록
**영문 식별자**: `cart`
**한글 표기**: 장바구니
**금지 표현**: ~~basket~~, ~~bag~~

#### 메타데이터
- **출처**: proj-A (2026-01-15), proj-B (2026-03-02), proj-C (2026-04-28)
- **확신도**: 3 (3개 프로젝트에서 채택)
- **인용 횟수**: 3
```

확신도가 높을수록 "이 용어는 검증됨" → 새 프로젝트에서 자동 채택 권장도 ↑.

## 원칙

| 원칙 | 설명 |
|------|------|
| **참고형 + 선택 채택** | 자동 상속 ❌. 새 프로젝트에서 사용자가 명시 선택 |
| **명시 업그레이드** | 자동 글로벌화 ❌. 프로젝트 종료 시 사용자가 명시 선택 |
| **충돌 보존** | 프로젝트 정의가 글로벌과 다르면 별칭 표기 또는 보류. 강제 통일 ❌ |
| **출처 추적** | 각 용어에 출처 프로젝트 + 날짜 메타데이터 |

## 다른 CLI 공유

Claude/Codex/Gemini가 같은 글로벌 사전을 공유할 수 있습니다 (mnemo 패턴과 동일).
구현은 `domain-dictionary` 스킬의 `references/global-sync.md`를 참조.

## 직접 편집

이 폴더의 파일들은 사용자가 직접 편집해도 됩니다. 자동 생성된 메타데이터(출처, 확신도)는 보존하되, 정의나 금지 표현은 자유롭게 수정 가능합니다.

## 도메인 추가

위에 나열되지 않은 도메인이 필요하면 자유롭게 새 파일을 추가하세요:
```
~/.claude/memory/domain-dictionaries/{your-domain}.md
```

스킬이 자동 인식합니다.
